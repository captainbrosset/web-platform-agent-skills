# Web audio

https://webaudio.github.io/web-audio-api/

## Web audio features

### AudioListener

An `AudioListener` object represents the virtual position and orientation of a person relative to sound sources, to spatialize audio in a Web Audio context.

**Spec:**
- <https://webaudio.github.io/web-audio-api/#AudioListener>

**Browser support:**
- Chrome: 52
- Chrome Android: 52
- Edge: 79
- Firefox: —
- Firefox for Android: —
- Safari: 14.1
- Safari on iOS: 14.5

### AudioListener setPosition() and setOrientation()

The `setPosition()` and `setOrientation()` methods of the `AudioListener` API set the virtual position and orientation of a person relative to sound sources, to spatialize audio in a Web Audio context.

**Spec:**
- <https://webaudio.github.io/web-audio-api/#AudioListener>

**Browser support:**
- Chrome: 14
- Chrome Android: 18
- Edge: 12
- Firefox: 25
- Firefox for Android: 25
- Safari: 6
- Safari on iOS: 6

**MDN:**
- [AudioListener: setOrientation() method](https://developer.mozilla.org/docs/Web/API/AudioListener/setOrientation)
- [AudioListener: setPosition() method](https://developer.mozilla.org/docs/Web/API/AudioListener/setPosition)

### AudioWorklet

The `AudioWorklet` API runs module code in a separate thread, specifically for non-blocking, low latency audio processing.

**Spec:**
- <https://webaudio.github.io/web-audio-api/#AudioWorklet>

**Browser support:**
- Chrome: 66
- Chrome Android: 66
- Edge: 79
- Firefox: 76
- Firefox for Android: 79
- Safari: 14.1
- Safari on iOS: 14.5

**MDN:**
- [AudioWorklet](https://developer.mozilla.org/docs/Web/API/AudioWorklet)

### OfflineAudioContext

The `OfflineAudioContext` API represents an audio processing graph that writes to an `AudioBuffer` instead of an output device.

**Spec:**
- <https://webaudio.github.io/web-audio-api/#OfflineAudioContext>

**Browser support:**
- Chrome: 35
- Chrome Android: 35
- Edge: 12
- Firefox: 25
- Firefox for Android: 25
- Safari: 14.1
- Safari on iOS: 14.5

**MDN:**
- [OfflineAudioContext](https://developer.mozilla.org/docs/Web/API/OfflineAudioContext)

### Web Audio

The Web Audio API creates graphs of audio nodes that handle sound inputs, synthesize sounds, apply effects, create visualizations, and output to audio devices.

**Spec:**
- <https://webaudio.github.io/web-audio-api/>

**Browser support:**
- Chrome: 35
- Chrome Android: 35
- Edge: 12
- Firefox: 25
- Firefox for Android: 25
- Safari: 14.1
- Safari on iOS: 14.5

**MDN:**
- [Web Audio API](https://developer.mozilla.org/docs/Web/API/Web_Audio_API)
